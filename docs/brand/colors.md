# Color Specifications

Complete color palette specifications for Atty Financial brand, including technical values, usage guidelines, accessibility requirements, and approved color combinations.

|----------|-------|

- Large blocks of color

## Usage Guidelines

### When to Use Primary Colors

**Black:**
- All body text and headings
- Primary navigation links
- Main call-to-action buttons
- Interactive elements on light backgrounds

**Sand:**
- Alternating section backgrounds
- Feature section backgrounds
- Page-wide background (default)
- Visual separation between content areas

**White:**
- Card and panel backgrounds
- Modal and dropdown backgrounds
- Form input backgrounds
- Content container backgrounds

**Gray:**
- Border lines and dividers
- Placeholder text in form inputs
- Disabled state indicators
- Subtle iconography

### When to Use Secondary Colors

**Green:**
- Success messages and confirmations
- Completed task indicators
- Positive status badges
- Growth/progress visualizations

**Periwinkle:**
- Information messages
- Neutral tag backgrounds
- Calendar highlights
- Info state indicators

**Melon:**
- Warning messages
- Attention-grabbing elements
- Caution indicators
- Feature callouts

**Yellow:**
- Decorative accents
- Featured content highlights
- Step progress indicators
- Callout boxes

### Color Proportion Guidelines

| Color Type | Recommended Usage |
|------------|------------------|
| Black | 60-70% (text, primary elements) |
| White | 20-30% (backgrounds, cards) |
| Sand | 30-40% (section backgrounds) |
| Gray | 10-15% (borders, subtle elements) |
| Secondary colors | 5-10% (accents, states) |

### Do's and Don'ts

**Do:**
✓ Use Black for all primary text on light backgrounds
✓ Use Sand for section backgrounds to create visual hierarchy
✓ Use White for content cards and containers
✓ Use Gray only for borders, dividers, and placeholder text
✓ Use secondary colors as accents only
✓ Test all color combinations for accessibility
✓ Maintain consistent contrast ratios throughout the interface

**Don't:**
✗ Use secondary colors for primary text
✗ Use Gray for body text on light backgrounds
✗ Create new colors outside the approved palette
✗ Use Black for large background areas
✗ Use multiple secondary colors together
✗ Rely on color alone to convey meaning
✗ Use colors that don't meet WCAG AA standards

## Accessibility Compliance

### WCAG 2.1 AA Requirements

- **Normal text:** Minimum 4.5:1 contrast ratio
- **Large text (18pt+ / 14pt bold):** Minimum 3:1 contrast ratio
- **UI components:** Minimum 3:1 contrast ratio
- **Focus indicators:** Visible on all interactive elements

### Testing Color Contrast

Use these tools to verify accessibility:
- [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
- [Chrome DevTools Lighthouse](https://developers.google.com/web/tools/lighthouse)
- [axe DevTools](https://www.deque.com/axe/)

### Focus States

All interactive elements must have visible focus states:

```css
/* Example focus state using brand colors */
button:focus-visible,
a:focus-visible,
input:focus-visible {
  outline: 2px solid var(--color-secondary-periwinkle);
  outline-offset: 2px;
}
```

### React Example

```tsx
import { tokens } from './brand-tokens';

function Button({ variant = 'primary', children }) {
  const styles = {
    primary: {
      backgroundColor: tokens.colors['primary-black'],
      color: tokens.colors['primary-white'],
    },
    success: {
      backgroundColor: tokens.colors['secondary-green'],
      color: tokens.colors['primary-black'],
    },
  };

  return <button style={styles[variant]}>{children}</button>;
}
```
